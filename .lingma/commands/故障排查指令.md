# TypeDom Signals Troubleshooting Specification

You are the debugging expert for TypeDom Signals. You diagnose issues, identify root causes, and provide proven solutions for common and edge-case problems.

## Context

- **Common Issues**: Update not triggering, memory leaks, infinite loops
- **Debug Tools**: Console logging, state inspection, dependency visualization
- **Recovery Strategies**: Force flush, state reset, dependency rebuild

## Diagnostic Framework

### Issue Classification

| Category                | Symptoms                              | Priority |
| ----------------------- | ------------------------------------- | -------- |
| Update Not Triggered    | Effect doesn't re-run                 | High     |
| Computed Stale          | Returns old value                     | High     |
| Effect Repeated         | Executes multiple times unexpectedly  | Medium   |
| Memory Leak             | Growing memory, effects persist       | High     |
| Performance Issue       | Slow updates,卡顿                      | Medium   |
| Circular Dependency     | Infinite loop detected                | Critical |
| Trigger Ineffective     | Manual trigger has no effect          | Medium   |

## Common Issues and Solutions

### 1. Update Not Triggered

#### Symptom
```typescript
const obj = signal({ a: 1 });
obj.get().a = 2; // Modification, but effect doesn't execute
```

#### Root Cause
Direct mutation of object/array internals doesn't change reference, so no update triggered.

#### Solution
```typescript
// ✅ Method 1: Replace entire object
obj.set({ ...obj.get(), a: 2 });

// ✅ Method 2: Use trigger manually
obj.get().a = 2;
trigger(obj);

// ✅ Method 3: Create new object
obj.set(Object.assign({}, obj.get(), { a: 2 }));
```

### 2. Computed Not Recomputing

#### Symptom
```typescript
const src = signal(0);
const comp = computed(() => src.get());

src.set(1);
console.log(comp.get()); // Still old value
```

#### Possible Causes

**Cause 1: Dependency Not Tracked**
```typescript
// ❌ WRONG: Access outside computed
const value = src.get();
const comp = computed(() => value);

// ✅ CORRECT: Access inside computed
const comp = computed(() => src.get());
```

**Cause 2: Flags State Corrupted**
```typescript
// Check flags state
console.log('Flags:', comp.flags);
// Should be Mutable|RecursedCheck or None

// Force refresh
trigger(comp);
```

### 3. Effect Repeated Execution

#### Symptom
```typescript
let count = 0;
effect(() => {
  count++;
  console.log('Executed', count);
});
// Executes multiple times instead of once
```

#### Possible Causes

**Cause 1: Signal Modified Inside Effect**
```typescript
// ❌ WRONG: Infinite loop
const src = signal(0);
effect(() => {
  console.log(src.get());
  src.set(src.get() + 1); // Triggers new effect
});

// ✅ CORRECT: Separate read/write
effect(() => {
  console.log(src.get());
});

// Modify externally
src.set(1);
```

**Cause 2: Nested Effect Not Cleaned**
```typescript
// ❌ WRONG: Inner effect not cleaned
effect(() => {
  if (condition.get()) {
    effect(() => {
      data.get();
    });
  }
});

// ✅ CORRECT: Use effectScope
const stopScope = effectScope(() => {
  effect(() => {
    condition.get();
  });
  
  effect(() => {
    data.get();
  });
});
```

### 4. Memory Leaks

#### Symptom
- Application slows down over time
- Effects still execute after disposal
- Memory usage grows continuously

#### Diagnosis
```typescript
// Check if effect properly cleaned
const stop = effect(() => {
  data.get();
});

// Call on component unmount
onUnmounted(() => {
  stop();
});

// Check subscriber count
console.log('Subs count:', data.subs);
```

#### Solution
```typescript
// ✅ Always cleanup effects
const stop = effect(() => {});
stop(); // When done

// ✅ Use effectScope for batch management
const scope = effectScope(() => {
  effect(() => {});
  effect(() => {});
});
scope(); // Cleanup all at once

// ✅ Use onScopeDispose in components
setup() {
  onScopeDispose(() => {
    cleanup();
  });
}
```

### 5. Dependency Order Issues

#### Symptom
```typescript
effect(() => {
  console.log(a.get(), b.get());
});

startBatch();
b.set(1);
a.set(1);
endBatch();

// Expected: 1, 1
// Actual: May be old values
```

#### Root Cause
Flush order at batch end may not match expectations.

#### Solution
```typescript
// ✅ Ensure correct dependency order
effect(() => {
  const aVal = a.get();
  const bVal = b.get();
  console.log(aVal, bVal);
});

// ✅ Use nested effects for ordering
effect(() => {
  a.get();
  effect(() => {
    b.get();
  });
});
```

### 6. Trigger Ineffective

#### Symptom
```typescript
arr.get().push(1);
trigger(arr); // No effect
```

#### Possible Causes

**Cause 1: No Dependency Access in Trigger**
```typescript
// ❌ WRONG: Trigger doesn't access signal
trigger(() => {});

// ✅ CORRECT: Access signal in trigger
trigger(() => {
  arr.get();
});
```

**Cause 2: Wrong Timing**
```typescript
// ❌ WRONG: Call during batch
startBatch();
arr.get().push(1);
trigger(arr); // May be overridden by endBatch
endBatch();

// ✅ CORRECT: Call outside batch
arr.get().push(1);
trigger(arr);
```

### 7. Performance Issues

#### Symptom
- Slow updates
- Noticeable lag
- High CPU usage

#### Diagnosis
```typescript
// Add performance monitoring
let computeCount = 0;
const expensive = computed(() => {
  const start = performance.now();
  const result = heavyComputation();
  console.log('Took', performance.now() - start, 'ms');
  computeCount++;
  return result;
});

// Check computation frequency
console.log('Compute count:', computeCount);
```

#### Optimization
```typescript
// ✅ Use caching
const cached = computed(() => {
  if (!needRecalc.get()) {
    return cachedValue; // Return cache
  }
  return expensiveCalculation();
});

// ✅ Batch updates
startBatch();
for (let i = 0; i < 100; i++) {
  signals[i].set(i);
}
endBatch();

// ✅ Avoid unnecessary computation
const lazy = computed(() => {
  if (!shouldCompute.get()) {
    return defaultValue; // No dependencies
  }
  return complexComputation(data.get());
});
```

## Debugging Techniques

### 1. Logging Method

#### Add Detailed Logs
```typescript
const src = signal(0);

effect(() => {
  console.group('Effect triggered');
  console.log('Current value:', src.get());
  console.trace(); // Print call stack
  console.groupEnd();
});
```

#### Monitor Signal Changes
```typescript
function monitorSignal<T>(sig: Signal<T>, name: string) {
  let lastValue = sig.get();
  
  effect(() => {
    const currentValue = sig.get();
    console.log(`${name} changed:`, {
      from: lastValue,
      to: currentValue,
      time: new Date().toISOString()
    });
    lastValue = currentValue;
  });
}

monitorSignal(count, 'count');
```


### 3. State Inspection

#### Check Node State
```typescript
function inspectNode(node: ReactiveNode, name: string) {
  console.log(`${name} inspection:`, {
    flags: node.flags,
    depsCount: countLinks(node.deps),
    subsCount: countLinks(node.subs),
    hasDepsTail: !!node.depsTail,
    hasSubsTail: !!node.subsTail
  });
}

function countLinks(link: Link | undefined): number {
  let count = 0;
  while (link) {
    count++;
    link = link.nextDep || link.nextSub;
  }
  return count;
}
```

### 4. Dependency Visualization

#### Print Dependency Tree
```typescript
function printDependencyTree(node: ReactiveNode, indent = 0): void {
  const prefix = '  '.repeat(indent);
  console.log(`${prefix}Node(flags=${node.flags})`);
  
  if (node.deps) {
    console.log(`${prefix}  Dependencies:`);
    let dep = node.deps;
    do {
      printDependencyTree(dep.dep, indent + 2);
      dep = dep.nextDep!;
    } while (dep);
  }
  
  if (node.subs) {
    console.log(`${prefix}  Subscribers:`);
    let sub = node.subs;
    do {
      printDependencyTree(sub.sub, indent + 2);
      sub = sub.nextSub!;
    } while (sub);
  }
}
```

## Common Error Patterns

### 1. Circular Dependency

#### Error Example
```typescript
const a = computed(() => b.get());
const b = computed(() => a.get()); // Circular!
```

#### Detection
```typescript
// Add cycle detection
let visiting = new Set<ReactiveNode>();

function updateComputed(c: ComputedNode): boolean {
  if (visiting.has(c)) {
    throw new Error('Circular dependency detected');
  }
  
  visiting.add(c);
  try {
    // ... update logic
  } finally {
    visiting.delete(c);
  }
}
```

### 2. Conditional Dependency

#### Error Example
```typescript
const cond = signal(true);
const src1 = signal(1);
const src2 = signal(2);

effect(() => {
  if (cond.get()) {
    console.log(src1.get());
  } else {
    console.log(src2.get());
  }
});

// Problem: Dependency changes when cond changes
```

#### Solution
```typescript
// ✅ Explicitly declare all possible dependencies
effect(() => {
  const c = cond.get();
  const v1 = src1.get();
  const v2 = src2.get();
  
  if (c) {
    console.log(v1);
  } else {
    console.log(v2);
  }
});
```

### 3. Async Update Issues

#### Error Example
```typescript
async function fetchData() {
  const data = await api.getData();
  signalData.set(data); // Async update
}

effect(() => {
  console.log(signalData.get());
});

fetchData();
```

#### Solution
```typescript
// ✅ Ensure async update in correct context
async function fetchData() {
  try {
    const data = await api.getData();
    startBatch();
    signalData.set(data);
    endBatch();
  } catch (error) {
    console.error('Failed:', error);
  }
}
```

## Recovery Strategies

### 1. Force Flush

```typescript
// Force flush all pending updates
function forceFlush() {
  while (getBatchDepth() > 0) {
    endBatch();
  }
}
```

### 2. Reset State

```typescript
// Reset signal to initial state
function resetSignal<T>(sig: Signal<T>, initialValue: T) {
  sig.pendingValue = initialValue;
  sig.currentValue = initialValue;
  sig.flags = ReactiveFlags.Mutable;
  trigger(sig);
}
```

### 3. Rebuild Dependencies

```typescript
// Rebuild effect dependencies
function rebuildEffect(stop: () => void, fn: () => void) {
  stop();
  return effect(fn);
}
```

## Error Handling Reference

| Error                          | Action                                                                                                      |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------- |
| Git rebase conflict            | Report to user, exit                                                                                        |
| `nx-cloud apply-locally` fails | Reject fix via MCP (`action: "REJECT"`), then attempt manual patch or exit                                  |
| MCP tool error                 | Retry once, if fails report to user                                                                         |
| Subagent spawn failure         | Retry once, if fails exit with error                                                                        |
| Decision script error          | Treat as `error` status, increment `no_progress_count`                                                      |
| No new CI Attempt detected     | If `--auto-fix-workflow`, try lockfile update; otherwise report with guidance                               |
| Lockfile auto-fix fails        | Report to user, exit with guidance to check CI logs                                                         |
