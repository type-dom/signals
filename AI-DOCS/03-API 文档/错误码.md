## 📖 概述

本文档说明 `@type-dom/signals` 的错误处理机制和错误码定义。

### 错误处理原则

1. **类型安全** - 使用 TypeScript 错误类型
2. **明确信息** - 错误信息清晰准确
3. **可恢复性** - 区分致命错误和可恢复错误
4. **性能优先** - 错误检查不影响正常路径性能

export class SignalError extends Error {
  constructor(message: string) {
    super(message);
    this.name = 'SignalError';
  }
}

export class CircularDependencyError extends SignalError {
  constructor() {
    super('Circular dependency detected');
    this.name = 'CircularDependencyError';
  }
}

export class EffectCleanupError extends SignalError {
  constructor(message: string) {
    super(`Effect cleanup error: ${message}`);
    this.name = 'EffectCleanupError';
  }
}

## 🚨 错误码列表

### 依赖追踪相关 (E001-E099)

| 错误码 | 错误名 | 说明 | 解决方案 |
|--------|--------|------|---------|
| **E001** | `CIRCULAR_DEPENDENCY` | 检测到循环依赖 | 重构代码避免循环 |
| **E002** | `MISSING_DEPENDENCY` | 访问未初始化的依赖 | 确保依赖已正确初始化 |
| **E003** | `INVALID_LINK` | Link 结构损坏 | 检查 link/unlink 逻辑 |

**E001 示例**:

```typescript
// ❌ 错误：循环依赖
const a = computed(() => b.get() + 1);
const b = computed(() => a.get() + 1);

// 运行时可能抛出:
// Error: E001 - Circular dependency detected
// at checkDirty (computed.ts:45)

// ✅ 解决：单向依赖
const a = signal(0);
const b = computed(() => a.get() + 1);
```

### 更新传播相关 (E100-E199)

| 错误码 | 错误名 | 说明 | 解决方案 |
|--------|--------|------|---------|
| **E100** | `PROPAGATION_FAILED` | 更新传播失败 | 检查依赖图完整性 |
| **E101** | `FLUSH_TIMEOUT` | flush 超时 | 检查是否有死锁 |
| **E102** | `BATCH_NESTED_OVERFLOW` | 批量更新嵌套过深 | 减少嵌套层级 |

**E100 示例**:

```typescript
// 如果 Link 链表断裂
signal.subs → Link1 → ??? → Link3

// 可能抛出:
// Error: E100 - Propagation failed
// at propagate (system.ts:78)
```

### 作用域管理相关 (E200-E299)

| 错误码 | 错误名 | 说明 | 解决方案 |
|--------|--------|------|---------|
| **E200** | `SCOPE_NOT_ACTIVE` | 在非活动作用域中操作 | 确保在 effectScope 内 |
| **E201** | `SCOPE_CLEANUP_FAILED` | 作用域清理失败 | 检查清理函数 |
| **E202** | `NESTED_SCOPE_CONFLICT` | 嵌套作用域冲突 | 避免作用域交叉 |

### 类型安全相关 (E300-E399)

| 错误码 | 错误名 | 说明 | 解决方案 |
|--------|--------|------|---------|
| **E300** | `TYPE_MISMATCH` | 类型不匹配 | 检查泛型约束 |
| **E301** | `UNDEFINED_VALUE` | 访问 undefined 值 | 添加类型守卫 |
| **E302** | `NULL_ACCESS` | 空值访问 | 使用可选链 |

## 🔧 错误处理机制

### 1. try-catch 包裹

```typescript
function runEffectSafely(effect: EffectNode): void {
  try {
    effect.fn();
  } catch (error) {
    // 记录错误但不中断程序
    console.error('Effect execution error:', error);
    
    // 可选择停止该 effect
    // stopEffect(effect);
  }
}
```

### 2. 错误边界模式

```typescript
class ErrorBoundary {
  private errors: Error[] = [];
  
  execute<T>(fn: () => T): T | undefined {
    try {
      return fn();
    } catch (error) {
      this.errors.push(error as Error);
      
      // 通知订阅者
      this.onError(error as Error);
      
      return undefined;
    }
  }
  
  onError(error: Error): void {
    // 子类可重写此方法
    console.error('Error boundary caught:', error);
  }
  
  getErrors(): Error[] {
    return [...this.errors];
  }
}

// 使用示例
const boundary = new ErrorBoundary();

effect(() => {
  const result = boundary.execute(() => {
    return riskyOperation(signal.get());
  });
  
  if (result !== undefined) {
    updateUI(result);
  }
});
```

### 3. 优雅降级

```typescript
const data = signal<Data | null>(null);
const error = signal<Error | null>(null);

async function fetchData() {
  try {
    const result = await api.getData();
    data.set(result);
    error.set(null);
  } catch (err) {
    error.set(err as Error);
    // data 保持原值，UI 显示旧数据
  }
}

// UI 自动响应变化
effect(() => {
  if (error.get()) {
    showError(error.get()!.message);
  } else {
    renderData(data.get());
  }
});
```

## 🛡️ 防御性编程

### 参数验证

```typescript
export function signal<T>(initialValue?: T): Signal<T> {
  // 验证参数
  if (arguments.length > 1) {
    throw new Error('signal accepts at most 1 argument');
  }
  
  return new Signal(initialValue);
}

export function computed<T>(
  getter: (previousValue?: T) => T,
  setter?: (newValue: T) => void
): Computed<T> {
  // 验证 getter 是函数
  if (typeof getter !== 'function') {
    throw new TypeError('getter must be a function');
  }
  
  // 验证 setter（如果提供）
  if (setter !== undefined && typeof setter !== 'function') {
    throw new TypeError('setter must be a function');
  }
  
  return new Computed(getter, setter);
}
```

### 状态验证

```typescript
function checkValidLink(link: Link): void {
  if (!link.dep || !link.sub) {
    throw new Error('E003 - Invalid link: missing dep or sub');
  }
  
  if (link.version <= 0) {
    throw new Error('E003 - Invalid link: invalid version');
  }
}

function checkNoCircularDependency(node: ReactiveNode): void {
  // 简化的循环检测
  const visited = new Set<ReactiveNode>();
  const stack = [node];
  
  while (stack.length > 0) {
    const current = stack.pop()!;
    
    if (visited.has(current)) {
      throw new Error('E001 - Circular dependency detected');
    }
    
    visited.add(current);
    
    // 添加到依赖栈
    if (current.deps) {
      let link = current.deps;
      do {
        stack.push(link.dep);
        link = link.nextDep!;
      } while (link !== current.deps);
    }
  }
}
```

## 📊 错误日志记录

### 结构化日志

```typescript
interface ErrorLog {
  timestamp: string;
  errorCode: string;
  errorMessage: string;
  stackTrace?: string;
  context: {
    signalName?: string;
    operation?: string;
    values?: any;
  };
}

const errorLogs: ErrorLog[] = [];

function logError(error: Error, context: any): void {
  const log: ErrorLog = {
    timestamp: new Date().toISOString(),
    errorCode: extractErrorCode(error),
    errorMessage: error.message,
    stackTrace: error.stack,
    context,
  };
  
  errorLogs.push(log);
  
  // 开发环境输出到控制台
  if (process.env.NODE_ENV === 'development') {
    console.error('[Signal Error]', log);
  }
}
```

### 错误统计

```typescript
interface ErrorStats {
  [errorCode: string]: {
    count: number;
    lastOccurrence: string;
    firstOccurrence: string;
  };
}

const errorStats: ErrorStats = {};

function trackError(errorCode: string): void {
  if (!errorStats[errorCode]) {
    errorStats[errorCode] = {
      count: 0,
      lastOccurrence: '',
      firstOccurrence: new Date().toISOString(),
    };
  }
  
  const stat = errorStats[errorCode];
  stat.count++;
  stat.lastOccurrence = new Date().toISOString();
}
```

## 📈 错误监控集成

### Sentry 集成

```typescript
import * as Sentry from '@sentry/browser';

// 全局错误处理
window.addEventListener('error', (event) => {
  if (event.message.includes('Signal')) {
    Sentry.captureException(event.error, {
      tags: {
        section: 'signals',
        error_code: extractErrorCode(event.error),
      },
    });
  }
});
```

### 自定义监控

```typescript
class SignalMonitor {
  private static instance: SignalMonitor;
  private errors: Map<string, number> = new Map();
  
  static getInstance(): SignalMonitor {
    if (!SignalMonitor.instance) {
      SignalMonitor.instance = new SignalMonitor();
    }
    return SignalMonitor.instance;
  }
  
  reportError(errorCode: string, details: any): void {
    const count = this.errors.get(errorCode) || 0;
    this.errors.set(errorCode, count + 1);
    
    // 超过阈值时告警
    if (count + 1 > 100) {
      this.sendAlert(errorCode, count + 1);
    }
  }
  
  private sendAlert(errorCode: string, count: number): void {
    // 发送告警通知
    console.warn(`Alert: ${errorCode} occurred ${count} times`);
  }
}
```
