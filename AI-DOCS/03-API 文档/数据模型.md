## 📖 概述

本文档详细说明 `@type-dom/signals` 中的核心数据模型，包括接口定义、类实现和数据结构。

### 数据模型层次

```
ReactiveNode (基础接口)
    ├── SignalNode<T> (信号节点)
    ├── ComputedNode<T, S> (计算节点)
    └── EffectNode (副作用节点)

Link (依赖链接)
ReactiveFlags (状态标志)
Stack<T> (栈结构)
```

## 🎯 核心接口

### 1. ReactiveNode

**作用**: 所有响应式节点的基类接口

```typescript
export interface ReactiveNode {
  // 依赖关系（我依赖谁）
  deps?: Link;        // 依赖链表的头节点
  depsTail?: Link;    // 依赖链表的尾节点
  
  // 订阅关系（谁依赖我）
  subs?: Link;        // 订阅链表的头节点
  subsTail?: Link;    // 订阅链表的尾节点
  
  // 状态标志
  flags: ReactiveFlags;
}
```

**字段详解**:

| 字段 | 类型 | 说明 | 示例场景 |
|------|------|------|---------|
| `deps` | `Link` | 第一个依赖 | computed 指向它使用的 signal |
| `depsTail` | `Link` | 最后一个依赖 | 快速添加到依赖链表末尾 |
| `subs` | `Link` | 第一个订阅者 | signal 指向使用它的 computed |
| `subsTail` | `Link` | 最后一个订阅者 | 快速添加到订阅链表末尾 |
| `flags` | `ReactiveFlags` | 状态标志集合 | Dirty, Pending, Watching |

**可视化结构**:

```
Signal Node (count)
├─ deps: undefined (signal 没有依赖)
├─ depsTail: undefined
├─ subs: Link1 → Link2 → Link3 (三个订阅者)
├─ subsTail: Link3
└─ flags: Mutable | Watching

Computed Node (doubled)
├─ deps: Link1 (指向 count)
├─ depsTail: Link1
├─ subs: Link1 (一个订阅者 effect)
├─ subsTail: Link1
└─ flags: Dirty
```

### 2. SignalNode

**作用**: 信号节点的扩展接口

```typescript
export interface SignalNode<T = any> extends ReactiveNode {
  currentValue?: T;     // 当前缓存的值
  pendingValue?: T;     // 待更新的值
}
```

**字段说明**:

| 字段 | 类型 | 说明 | 更新时机 |
|------|------|------|---------|
| `currentValue` | `T` | 当前已知的最新值 | flush() 后更新 |
| `pendingValue` | `T` | 设置但尚未 flush 的值 | set() 时更新 |

**生命周期**:

```
1. 创建: currentValue = initialValue, pendingValue = initialValue
2. set(newValue): pendingValue = newValue, flags |= Dirty
3. propagate(): 标记订阅者为 Dirty/Pending
4. get(): 如果 Dirty，触发 update，同步 currentValue 和 pendingValue
5. flush(): currentValue = pendingValue
```

### 3. ComputedNode

**作用**: 计算节点的扩展接口

```typescript
export interface ComputedNode<T = any, S = T> extends SignalNode<T> {
  value: T | undefined;           // 缓存的计算结果
  getter: (previousValue?: T) => T; // 计算函数
  setter?: (newValue: S) => void;   // 可选的 setter
}
```

**字段说明**:

| 字段 | 类型 | 说明 | 特性 |
|------|------|------|------|
| `value` | `T` | 缓存的计算结果 | 惰性求值，访问时才计算 |
| `getter` | `Function` | 计算函数 | 接收 previousValue |
| `setter` | `Function` | 可选 setter | 使 computed 可写 |

**计算流程**:

```typescript
computed.get() {
  // 1. 检查是否需要重新计算
  if (flags & Dirty) {
    // 2. 调用 getter，传入旧值
    value = getter(previousValue);
    
    // 3. 清除 Dirty 标志
    flags &= ~Dirty;
    
    // 4. 如果有 setter 且值变化
    if (setter && value !== previousValue) {
      setter(value);
    }
  }
  
  return value;
}
```

### 4. EffectNode

**作用**: 副作用节点接口

```typescript
export interface EffectNode extends ReactiveNode {
  fn(): void;              // 副作用函数
  running?: boolean;       // 是否正在执行
}
```

**字段说明**:

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `fn` | `Function` | 副作用函数 | 必填 |
| `running` | `boolean` | 执行状态标志 | false |

**执行流程**:

```typescript
function runEffect(effect: EffectNode) {
  // 1. 防止重入
  if (effect.running) return;
  
  effect.running = true;
  
  try {
    // 2. 清除旧依赖
    unlink(effect);
    
    // 3. 设置 activeSub，使 effect 内访问 signal 时建立依赖
    const prevActiveSub = activeSub;
    activeSub = effect;
    
    try {
      // 4. 执行副作用函数
      effect.fn();
    } finally {
      // 5. 恢复 activeSub
      activeSub = prevActiveSub;
    }
  } finally {
    effect.running = false;
  }
}
```

## 🔗 连接结构

### 5. Link

**作用**: 连接两个 ReactiveNode 的双向链表节点

```typescript
export interface Link {
  version: number;              // 版本号，用于验证有效性
  dep: ReactiveNode;            // 被依赖的节点
  sub: ReactiveNode;            // 订阅者节点
  prevSub: Link | undefined;    // 前一个订阅者
  nextSub: Link | undefined;    // 后一个订阅者
  prevDep: Link | undefined;    // 前一个依赖
  nextDep: Link | undefined;    // 后一个依赖
}
```

**字段图解**:

```
从 Signal 角度（订阅链表）:
Signal.subs → Link1 → Link2 → Link3
                ↓sub     ↓sub     ↓sub
              Comp1   Comp2   Comp3

从 Computed 角度（依赖链表）:
Computed.deps → Link1 → Link2
                 ↓dep     ↓dep
               Signal1  Signal2
```

**双向链接**:

```typescript
// Link1 在 Signal.subs 链表中
Link1.prevSub = undefined;      // 第一个订阅者
Link1.nextSub = Link2;          // 下一个订阅者是 Link2

// Link1 在 Computed.deps 链表中
Link1.prevDep = undefined;      // 第一个依赖
Link1.nextDep = Link2;          // 下一个依赖是 Link2
```

**版本号机制**:

```typescript
// 每次依赖收集周期递增
let cycle = 0;

function collectDependencies() {
  cycle++; // 新周期
  
  // 创建或更新 Link 时设置版本号
  link.version = cycle;
  
  // 检查 Link 是否有效
  if (link.version !== cycle) {
    // 过期，需要重新收集
  }
}
```

## 🚩 状态标志

### 6. ReactiveFlags

**作用**: 位运算枚举，表示节点状态

```typescript
export const enum ReactiveFlags {
  None = 0,           // 000000 - 无状态
  Mutable = 1,        // 000001 - 可变/已变异
  Watching = 2,       // 000010 - 正在监听
  RecursedCheck = 4,  // 000100 - 递归检查中
  Recursed = 8,       // 001000 - 已递归
  Dirty = 16,         // 010000 - 脏数据需更新
  Pending = 32,       // 100000 - 待处理
}
```

**位运算操作**:

```typescript
// 检查标志
if (node.flags & ReactiveFlags.Dirty) {
  // 需要更新
}

// 设置标志
node.flags |= ReactiveFlags.Watching;

// 清除标志
node.flags &= ~ReactiveFlags.Pending;

// 组合标志
const flags = ReactiveFlags.Mutable | ReactiveFlags.Dirty; // 17
```

**标志含义详解**:

| 标志 | 二进制 | 说明 | 设置时机 | 清除时机 |
|------|--------|------|---------|---------|
| `Mutable` | 000001 | 节点可以变异 | signal 初始化 | - |
| `Watching` | 000010 | 有活跃的 effect | effect 首次执行 | effect 停止 |
| `RecursedCheck` | 000100 | 正在检查依赖 | checkDirty 开始 | checkDirty 结束 |
| `Recursed` | 001000 | 已检查过依赖 | checkDirty 完成 | 下次检查 |
| `Dirty` | 010000 | 值已过时 | set() 或依赖更新 | get() 后重新计算 |
| `Pending` | 100000 | 等待传播更新 | propagate() | flush() |

**状态转换图**:

```
创建
  ↓
[Mutable] ←───┐
  ↓           │
set()        │
  ↓           │
[Mutable + Dirty] ──→ propagate()
  ↓                    ↓
[Mutable + Dirty + Pending]
  ↓
get() / update()
  ↓
[Mutable] ────────┘
```

## 📚 辅助数据结构

### 7. Stack<T>

**作用**: 用于非递归传播的栈结构

```typescript
interface Stack<T> {
  value: T;
  prev: Stack<T> | undefined;
}
```

**使用场景**:

```typescript
function propagate(link: Link): void {
  let stack: Stack<Link | undefined> | undefined;
  let next = link.nextSub;
  
  do {
    // ... 处理当前节点
    
    // 压栈
    if (hasMoreSubscribers) {
      stack = { value: next, prev: stack };
      next = nextSub;
      continue;
    }
    
    // 出栈
    if (next === undefined) {
      if (stack === undefined) break;
      next = stack.value;
      stack = stack.prev;
    }
  } while (true);
}
```

**优势**:

- ✅ 避免递归调用栈溢出
- ✅ 利用堆空间，理论上无限深度
- ✅ O(n) 时间复杂度

## 🎯 实际应用示例

### 示例 1: 简单依赖链

```typescript
const count = signal(0);
const doubled = computed(() => count.get() * 2);
const quadrupled = computed(() => doubled.get() * 2);

effect(() => {
  console.log(quadrupled.get());
});

// 数据结构:
// count.subs → Link(doubled)
// doubled.deps → Link(count)
// doubled.subs → Link(quadrupled)
// quadrupled.deps → Link(doubled)
// quadrupled.subs → Link(effect)
```

### 示例 2: 多依赖 computed

```typescript
const a = signal(1);
const b = signal(2);
const sum = computed(() => a.get() + b.get());

// 数据结构:
// sum.deps → Link(a) → Link(b)
// a.subs → Link(sum)
// b.subs → Link(sum)
```

### 示例 3: 动态依赖

```typescript
const condition = signal(true);
const source1 = signal('A');
const source2 = signal('B');

const dynamic = computed(() => {
  return condition.get() ? source1.get() : source2.get();
});

// condition = true 时:
// dynamic.deps → Link(condition) → Link(source1)

// condition = false 时:
// dynamic.deps → Link(condition) → Link(source2)
```

## 📊 内存布局优化

### V8 引擎优化

```typescript
// ✅ 推荐：隐藏类优化
class Signal {
  pendingValue: any;    // 固定顺序
  currentValue: any;    // 固定顺序
  subs: Link | undefined;
  subsTail: Link | undefined;
  flags: number;
}

// ❌ 不推荐：动态添加属性
const obj = {};
obj.pendingValue = 0;   // 破坏隐藏类
obj.currentValue = 0;
```

### 对象池模式

```typescript
// Link 对象池
const linkPool: Link[] = [];

function acquireLink(): Link {
  return linkPool.pop() || createNewLink();
}

function releaseLink(link: Link) {
  // 重置属性
  link.version = 0;
  link.dep = null!;
  link.sub = null!;
  link.prevSub = undefined;
  link.nextSub = undefined;
  link.prevDep = undefined;
  link.nextDep = undefined;
  
  linkPool.push(link);
}
```
