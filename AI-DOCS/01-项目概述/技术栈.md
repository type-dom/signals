```yaml

  - TypeScript ESLint (TS 规则)
```

## 💻 TypeScript 5.7+

### 1.1 版本选择理由

**为什么选择 TypeScript 5.7+?**

1. **更强大的类型推断**
   - const 参数推断改进
   - 泛型约束优化
   - 更好的联合类型处理

2. **性能提升**
   - 编译速度提升 30%
   - 增量编译优化
   - 内存占用降低

3. **新特性支持**
   - Decorator 元数据
   - Import Attributes
   - 更严格的检查选项

### 1.3 关键配置说明

#### 严格模式 (`strict: true`)

```typescript
// ✅ 启用严格模式后
const signal = <T>(value: T): Signal<T> => {
  // value 不会是 undefined (除非显式声明)
  return new Signal(value);
};

// ❌ 这会报错
const badSignal = (value) => { 
  // Error: Parameter 'value' implicitly has an 'any' type
};
```

#### 模块解析 (`moduleResolution: "node"`)

```typescript
// 支持 Node.js 风格的导入
import { signal } from '@type-dom/signals';
import { createReactiveSystem } from './system';
```

#### 隔离模块 (`isolatedModules: true`)

```typescript
// 确保每个文件可独立编译
// 这对 Rollup 打包很重要
export const enum ReactiveFlags {
  None = 0,
  Mutable = 1,
  // ...
}
```

## 🔧 Rollup 4.x

### 2.1 版本选择理由

**为什么选择 Rollup 4.x?**

1. **Tree Shaking 最优**
   - 自动移除未使用代码
   - 保持最小包体积

2. **多格式输出**
   - ESM (.mjs)
   - CJS (.cjs)
   - UMD (.umd.js)
   - IIFE

3. **插件生态丰富**
   - 完整的 TypeScript 支持
   - 各种预处理器支持
   - 代码压缩和优化

### 2.2 rollup.config.ts 配置

```typescript
import typescript from '@rollup/plugin-typescript';
import resolve from '@rollup/plugin-node-resolve';
import commonjs from '@rollup/plugin-commonjs';
import terser from '@rollup/plugin-terser';
import { defineConfig } from 'rollup';

export default defineConfig({
  input: 'src/index.ts',
  output: [
    {
      file: 'dist/signals.esm.js',
      format: 'esm',
      sourcemap: true,
    },
    {
      file: 'dist/signals.cjs.js',
      format: 'cjs',
      sourcemap: true,
    },
    {
      file: 'dist/signals.umd.js',
      format: 'umd',
      name: 'TypeDomSignals',
      sourcemap: true,
    },
  ],
  plugins: [
    resolve(), // 解析 node_modules
    commonjs(), // 转换 CommonJS 为 ES Module
    typescript({
      tsconfig: './tsconfig.json',
      declaration: true,
      declarationDir: './dist',
    }),
    terser({
      format: {
        comments: false, // 移除注释
      },
      compress: {
        drop_console: true, // 生产环境移除 console
        pure_funcs: ['console.log'],
      },
    }),
  ],
  external: [], // 外部依赖
  watch: {
    include: 'src/**',
    exclude: 'node_modules/**',
  },
});
```

### 2.4 输出格式说明

#### ESM (ECMAScript Module)

```javascript
// signals.esm.js
import { signal, computed, effect } from '@type-dom/signals';

export { signal, computed, effect };
export default { signal, computed, effect };
```

#### CJS (CommonJS)

```javascript
// signals.cjs.js
const { signal, computed, effect } = require('@type-dom/signals');

module.exports = { signal, computed, effect };
```

#### UMD (Universal Module Definition)

```javascript
// signals.umd.js
(function(root, factory) {
  if (typeof define === 'function' && define.amd) {
    define([], factory);
  } else if (typeof module === 'object' && module.exports) {
    module.exports = factory();
  } else {
    root.TypeDomSignals = factory();
  }
}(this, function() {
  return { signal, computed, effect };
}));
```

## 🧪 Vitest 3.x

### 3.1 版本选择理由

**为什么选择 Vitest 3.x?**

1. **极速**
   - 基于 Vite，热更新支持
   - 并行测试执行
   - 比 Jest 快 2-3 倍

2. **兼容 Jest**
   - API 与 Jest 高度兼容
   - 迁移成本低
   - 支持 Jest 快照

3. **内置覆盖率**
   - V8 引擎覆盖率统计
   - 实时覆盖率报告
   - 阈值检查

### 3.2 vitest.config.ts 配置

```typescript
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    // 全局配置
    globals: true, // 全局注入 describe, it, expect
    pool: 'threads', // 多线程执行
    testTimeout: 30000, // 超时时间
    
    // 覆盖率配置
    coverage: {
      provider: 'v8', // 使用 V8 引擎覆盖率
      reporter: ['text', 'json', 'html'], // 输出格式
      threshold: {
        global: {
          branches: 90, // 分支覆盖率 ≥ 90%
          functions: 90, // 函数覆盖率 ≥ 90%
          lines: 90, // 行覆盖率 ≥ 90%
          statements: 90, // 语句覆盖率 ≥ 90%
        },
      },
      include: ['src/**/*.ts'],
      exclude: ['src/index.ts'], // 导出文件不检查
    },
    
    // 文件匹配
    include: ['tests/**/*.test.ts'],
    
    // 环境配置
    environment: 'node',
    
    // 设置文件
    setupFiles: ['./tests/setup.ts'],
  },
});
```

### 3.3 测试示例

```typescript
// tests/signal.test.ts
import { describe, it, expect, beforeEach } from 'vitest';
import { signal, computed, effect } from '../src/index';

describe('Signal', () => {
  let countSignal: ReturnType<typeof signal<number>>;

  beforeEach(() => {
    countSignal = signal(0);
  });

  it('should create signal with initial value', () => {
    expect(countSignal.get()).toBe(0);
  });

  it('should update value on set', () => {
    countSignal.set(5);
    expect(countSignal.get()).toBe(5);
  });

  it('should not trigger effect for same value', () => {
    let callCount = 0;
    effect(() => {
      callCount++;
      countSignal.get();
    });
    
    expect(callCount).toBe(1);
    countSignal.set(0); // 相同值
    expect(callCount).toBe(1); // 不会重新执行
  });

  it('should track dependencies in computed', () => {
    const doubled = computed(() => countSignal.get() * 2);
    expect(doubled.get()).toBe(0);
    
    countSignal.set(5);
    expect(doubled.get()).toBe(10);
  });
});
```
