## 🎯 项目简介

### 什么是 TypeDOM Signals？

`@type-dom/signals` 是一个基于 **Push-Pull 混合模型**的高性能响应式信号库，采用 TypeScript 5.7+ 编写，提供细粒度的状态管理能力。

**核心理念**：
- ✅ **极致性能** - 非递归传播算法，O(1) 依赖追踪
- ✅ **类型安全** - 完整的 TypeScript 类型推断
- ✅ **零依赖** - 无运行时依赖，保持轻量
- ✅ **开发者友好** - 简洁直观的 API 设计

### 快速示例

```typescript
import { signal, computed, effect } from '@type-dom/signals';

// 1. 创建响应式信号
const count = signal(0);
const multiplier = signal(2);

// 2. 创建计算属性（自动追踪依赖）
const doubled = computed(() => count.get() * multiplier.get());

// 3. 创建副作用（自动重新执行）
effect(() => {
  console.log(`${count} × ${multiplier} = ${doubled.get()}`);
});
// 输出："0 × 2 = 0"

// 4. 更新信号，自动触发更新
count.set(5);
// 输出："5 × 2 = 10"

multiplier.set(3);
// 输出："5 × 3 = 15"
```

## 🏗️ 设计原则

### 1. 性能优先（Performance First）

**设计决策**：
- ✅ 使用 **非递归传播算法** 避免栈溢出
- ✅ 采用 **惰性求值** 减少不必要计算
- ✅ 实现 **批量更新** 合并多次变更
- ✅ 使用 **位运算标志** 提升状态检查性能

**性能指标**：
```
依赖追踪开销：< 0.5μs
更新传播延迟：< 2.3μs
内存占用：< 100KB
Bundle 大小：< 2KB (gzip)
```

### 2. 类型安全（Type Safety）

**设计决策**：
- ✅ 100% TypeScript 编写
- ✅ 严格模式配置
- ✅ 完整的泛型支持
- ✅ 精确的类型推断

**示例**：
```typescript
// ✅ 类型安全
const count: Signal<number> = signal(0);
count.set(5);        // OK
count.set('text');   // ❌ TypeScript Error

// ✅ 泛型支持
const user = signal<User>({ id: 1, name: 'John' });
user.get().name;     // 类型推断为 string
```

### 3. 简洁 API（Simple API）

**设计决策**：
- ✅ 最小化 API 表面
- ✅ 符合直觉的命名
- ✅ 一致的行为模式
- ✅ 零学习曲线

**API 对比**：
```typescript
// Vuex (复杂)
store.commit('increment');
store.state.count;

// TypeDOM Signals (简单)
count.set(count.get() + 1);
count.get();
```

### 4. 零依赖（Zero Dependencies）

**设计决策**：
- ✅ 无运行时依赖
- ✅ 纯函数实现
- ✅ 自包含的核心逻辑
- ✅ 易于集成和部署

## 🌟 应用场景

### 场景 1：前端框架集成

**适用框架**：React, Vue, Solid, Preact

```typescript
// React 集成示例
import { signal, computed } from '@type-dom/signals';

const count = signal(0);
const doubled = computed(() => count.get() * 2);

function Counter() {
  const [value, setValue] = useState(0);
  
  // 连接 Signals 和 React 状态
  effect(() => {
    setValue(count.get());
  });
  
  return (
    <div>
      <p>Count: {value}</p>
      <p>Doubled: {doubled.get()}</p>
      <button onClick={() => count.set(count.get() + 1)}>
        Increment
      </button>
    </div>
  );
}
```

### 场景 2：状态管理

**替代方案**：Redux, MobX, Zustand

```typescript
// 全局状态管理
const userStore = {
  name: signal<string>(''),
  email: signal<string>(''),
  isAuthenticated: computed(() => {
    return userStore.name.get().length > 0;
  }),
};

// 业务逻辑
function login(name: string, email: string) {
  userStore.name.set(name);
  userStore.email.set(email);
  // 自动触发所有依赖更新
}
```

### 场景 3：响应式表单

```typescript
// 表单验证
const email = signal('');
const error = computed(() => {
  const value = email.get();
  if (!value) return '必填项';
  if (!/^\S+@\S+\.\S+$/.test(value)) return '格式错误';
  return null;
});

// 自动验证
effect(() => {
  console.log('Validation:', error.get());
});
```

### 场景 4：数据可视化

```typescript
// 图表数据更新
const dataPoints = signal<DataPoint[]>([]);
const average = computed(() => {
  const points = dataPoints.get();
  return points.reduce((sum, p) => sum + p.value, 0) / points.length;
});

// 实时更新数据
setInterval(() => {
  const newPoint = fetchNewData();
  dataPoints.set([...dataPoints.get(), newPoint]);
  // 平均值自动重新计算
}, 1000);
```
