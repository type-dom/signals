## 🎯 故事一：React 开发者的表单优化

### 用户画像

**姓名**: 张明  
**职位**: 高级前端工程师  
**公司**: 某电商企业  
**技术栈**: React + TypeScript

### 痛点

```typescript
// ❌ 问题代码：性能瓶颈
function ComplexForm() {
  const [formData, setFormData] = useState({
    firstName: '',
    lastName: '',
    email: '',
    phone: '',
    address: '',
    // ... 20+ 字段
  });
  
  // 每次输入都重新渲染整个表单
  const handleChange = (field: string, value: string) => {
    setFormData(prev => ({ ...prev, [field]: value }));
  };
  
  return (
    <form>
      <Input value={formData.firstName} onChange={v => handleChange('firstName', v)} />
      <Input value={formData.lastName} onChange={v => handleChange('lastName', v)} />
      {/* ... 所有字段都会重新渲染 */}
    </form>
  );
}
```

**问题**:
- 每次输入触发整个表单重渲染
- 大型表单卡顿严重
- 验证逻辑复杂难以维护

### 解决方案

```typescript
// ✅ 使用 Signals 优化
import { signal, computed, effect } from '@type-dom/signals';

function ComplexForm() {
  // 每个字段独立信号
  const firstName = signal('');
  const lastName = signal('');
  const email = signal('');
  
  // 计算属性：自动验证
  const isEmailValid = computed(() => {
    const value = email.get();
    return /^\S+@\S+\.\S+$/.test(value);
  });
  
  const canSubmit = computed(() => {
    return firstName.get().length > 0 && 
           lastName.get().length > 0 && 
           isEmailValid.get();
  });
  
  // 副作用：实时更新错误提示
  effect(() => {
    if (!isEmailValid.get() && email.get().length > 0) {
      showError('邮箱格式不正确');
    }
  });
  
  // 只渲染一次，后续细粒度更新
  return (
    <form>
      <InputSignal signal={firstName} label="名" />
      <InputSignal signal={lastName} label="姓" />
      <InputSignal signal={email} label="邮箱" />
      
      {!isEmailValid.get() && (
        <Error>邮箱格式不正确</Error>
      )}
      
      <button disabled={!canSubmit.get()}>
        提交
      </button>
    </form>
  );
}
```

**收益**:
- ✅ 只渲染变化的字段
- ✅ 验证自动执行
- ✅ 性能提升 60%

### 传统方案的问题

export const useUserStore = defineStore('user', {
  state: () => ({
    name: '',

  }),

    displayName(state) {

### Signals 方案

```typescript
// ✅ 简洁的 Signals 实现
// stores/user.ts
export const userStore = {
  name: signal(''),
  email: signal(''),
  preferences: signal<UserPreferences>({}),
  
  // 计算属性
  displayName: computed(() => {
    return userStore.name.get() || '匿名用户';
  }),
  
  isAuthenticated: computed(() => {
    return userStore.email.get().length > 0;
  }),
  
  // 方法
  login(email: string) {
    this.email.set(email);
    // 自动触发所有依赖更新
  },
  
  logout() {
    this.name.set('');
    this.email.set('');
  },
};

// 组件中使用
function UserProfile() {
  return (
    <div>
      <h1>{userStore.displayName.get()}</h1>
      {userStore.isAuthenticated.get() ? (
        <button onClick={() => userStore.logout()}>退出</button>
      ) : (
        <button onClick={() => userStore.login('user@example.com')}>登录</button>
      )}
    </div>
  );
}
```

**优势**:
- ✅ 代码量减少 70%
- ✅ 无需注册 store
- ✅ 类型推断完整

**开源项目**: SolidUI 维护者

### 需求

开发一套高性能、细粒度更新的 UI 组件库。

### 实现 Button 组件

```typescript
// ✅ 使用 Signals 的 Button 组件
interface ButtonProps {
  variant?: 'primary' | 'secondary';
  disabled?: boolean;
  loading?: boolean;
  children: JSX.Element;
}

export function Button(props: ButtonProps) {
  // 响应式 props
  const variant = signal(props.variant || 'primary');
  const disabled = signal(props.disabled || false);
  const loading = signal(props.loading || false);
  
  // 计算样式
  const className = computed(() => {
    const base = 'btn';
    const v = variant.get();
    const d = disabled.get();
    const l = loading.get();
    
    return `${base} ${base}-${v}${d ? ' disabled' : ''}${l ? ' loading' : ''}`;
  });
  
  // 计算是否可点击
  const isClickable = computed(() => {
    return !disabled.get() && !loading.get();
  });
  
  // 副作用：日志记录
  effect(() => {
    console.log(`Button state: ${variant.get()}, ${disabled.get()}, ${loading.get()}`);
  });
  
  return (
    <button 
      class={className.get()}
      disabled={!isClickable.get()}
    >
      {props.children}
      {loading.get() && <Spinner />}
    </button>
  );
}

// 使用示例
function App() {
  const [loading, setLoading] = createSignal(false);
  
  return (
    <Button 
      variant="primary"
      loading={loading()}
      onClick={() => setLoading(true)}
    >
      提交
    </Button>
  );
}
```

**特点**:
- ✅ 样式自动计算
- ✅ 只更新变化的 DOM
- ✅ 易于组合和扩展

## 🎯 故事四：游戏开发者的实时数据更新

**姓名**: 赵雷  
**职位**: 游戏前端开发  
**公司**: 某游戏公司  
**项目**: 网页策略游戏

### 需求

- 实时更新玩家资源（金币、木材、铁矿）
- 动态显示建筑状态
- 战斗数值即时计算

### 实现

```typescript
// 游戏状态管理
class GameState {
  // 资源
  gold = signal(1000);
  wood = signal(500);
  iron = signal(300);
  
  // 建筑
  barracks = signal({ level: 1, training: false });
  farm = signal({ level: 3, producing: true });
  
  // 计算属性：总资源价值
  totalResources = computed(() => {
    return this.gold.get() * 1 + 
           this.wood.get() * 0.5 + 
           this.iron.get() * 1.5;
  });
  
  // 计算属性：最大人口
  maxPopulation = computed(() => {
    return this.farm.get().level * 10;
  });
  
  // 方法：采集资源
  gatherResource(type: 'gold' | 'wood' | 'iron', amount: number) {
    const current = this[type].get();
    this[type].set(current + amount);
    // 自动更新 totalResources
  }
  
  // 方法：升级建筑
  upgradeBuilding(building: 'barracks' | 'farm') {
    const cost = building === 'barracks' ? 200 : 150;
    
    if (this.gold.get() >= cost) {
      this.gold.update(g => g - cost);
      this[building].update(b => ({ ...b, level: b.level + 1 }));
    }
  }
}

const game = new GameState();

// UI 自动响应变化
effect(() => {
  updateResourceUI({
    gold: game.gold.get(),
    wood: game.wood.get(),
    iron: game.iron.get(),
    total: game.totalResources.get(),
  });
});

effect(() => {
  updateBuildingUI({
    barracks: game.barracks.get(),
    farm: game.farm.get(),
    maxPop: game.maxPopulation.get(),
  });
});
```

**效果**:
- ✅ 每秒 60 帧更新无压力
- ✅ 自动计算衍生数据
- ✅ 代码清晰易维护

## 🎯 故事五：数据可视化工程师的仪表盘

### 用户画像

**姓名**: 陈雪  
**职位**: 数据可视化工程师  
**行业**: 金融科技  
**场景**: 实时股票行情监控

### 需求

- 实时显示多支股票价格
- 自动计算涨跌幅、均线等指标
- 条件格式化（涨绿跌红）
- 性能要求极高

### 实现

```typescript
// 股票数据模型
interface Stock {
  symbol: string;
  price: number;
  change: number;
  changePercent: number;
}

// 单个股票信号
function createStockSignal(symbol: string) {
  const stock = signal<Stock>({
    symbol,
    price: 0,
    change: 0,
    changePercent: 0,
  });
  
  // 计算属性：格式化价格
  const formattedPrice = computed(() => {
    return `$${stock.get().price.toFixed(2)}`;
  });
  
  // 计算属性：涨跌颜色
  const priceColor = computed(() => {
    const change = stock.get().change;
    return change >= 0 ? '#00c853' : '#ff3d00'; // 涨绿跌红
  });
  
  // 计算属性：涨跌幅文本
  const changeText = computed(() => {
    const s = stock.get();
    const sign = s.change >= 0 ? '+' : '';
    return `${sign}${s.change.toFixed(2)} (${sign}${s.changePercent.toFixed(2)}%)`;
  });
  
  return { stock, formattedPrice, priceColor, changeText };
}

// 多支股票
const stocks = {
  AAPL: createStockSignal('AAPL'),
  GOOGL: createStockSignal('GOOGL'),
  MSFT: createStockSignal('MSFT'),
};

// 实时更新（WebSocket 推送）
websocket.on('stock-update', (data) => {
  const { symbol, price } = data;
  stocks[symbol as keyof typeof stocks].stock.update(s => ({
    ...s,
    price,
    change: price - s.price,
    changePercent: ((price - s.price) / s.price) * 100,
  }));
});

// UI 渲染
function StockRow({ symbol }: { symbol: string }) {
  const { stock, formattedPrice, priceColor, changeText } = stocks[symbol as keyof typeof stocks];
  
  return (
    <tr>
      <td>{stock.get().symbol}</td>
      <td style={{ color: priceColor.get() }}>
        {formattedPrice.get()}
      </td>
      <td style={{ color: priceColor.get() }}>
        {changeText.get()}
      </td>
    </tr>
  );
}
```

**性能指标**:
- ✅ 每秒处理 1000+ 次更新
- ✅ 只重绘变化的单元格
- ✅ CPU 占用降低 40%

## 🎯 故事六：全栈开发者的 SSR 应用

### 用户画像

**姓名**: 刘洋  
**职位**: 全栈开发工程师  
**框架**: Next.js / Nuxt.js

### 需求

在服务端渲染 (SSR) 环境中使用 signals，同时支持客户端水合 (hydration)。

### 实现

```typescript
// lib/ssr-signal.ts
import { signal as createSignal } from '@type-dom/signals';

export function signal<T>(initialValue: T, key?: string): Signal<T> {
  // SSR 环境：从全局获取或创建
  if (typeof window === 'undefined') {
    globalThis.__SSR_SIGNALS__ = globalThis.__SSR_SIGNALS__ || {};
    
    if (key && globalThis.__SSR_SIGNALS__[key]) {
      return globalThis.__SSR_SIGNALS__[key];
    }
    
    const sig = createSignal(initialValue);
    if (key) {
      globalThis.__SSR_SIGNALS__[key] = sig;
    }
    return sig;
  }
  
  // 客户端：直接使用或水合
  if (key && window.__SSR_SIGNALS__?.[key]) {
    const hydrated = createSignal(window.__SSR_SIGNALS__[key]);
    delete window.__SSR_SIGNALS__[key];
    return hydrated;
  }
  
  return createSignal(initialValue);
}

// 页面组件
export async function getServerSideProps() {
  const userData = await fetchUserData();
  
  return {
    props: {
      initialData: userData,
    },
  };
}

function Page({ initialData }: { initialData: UserData }) {
  // SSR + Hydration
  const user = signal(initialData, 'user');
  
  // 客户端交互
  const handleUpdate = () => {
    user.set({ ...user.get(), name: 'New Name' });
  };
  
  return (
    <div>
      <h1>{user.get().name}</h1>
      <button onClick={handleUpdate}>更新</button>
    </div>
  );
}
```
