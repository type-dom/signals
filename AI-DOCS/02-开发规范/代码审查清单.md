## 🔧 Signal 相关审查

### Signal 实现

```typescript
// 审查要点
class Signal<T> {
  // ✓ 属性数量合理（≤ 5 个）
  pendingValue: T | undefined;
  currentValue: T | undefined;
  subs?: Link;
  subsTail?: Link;
  flags: ReactiveFlags;
  
  // ✓ get() 方法包含依赖追踪
  get() {
    if (activeSub !== undefined) {
      linkReactiveNode(this, activeSub);
    }
    return this.pendingValue!;
  }
  
  // ✓ set() 方法包含相同值检查
  set(value: T) {
    if (this.pendingValue === value) {
      return; // 跳过相同值
    }
    this.pendingValue = value;
    this.flags |= ReactiveFlags.Dirty;
    if (this.subs !== undefined) {
      shallowPropagate(this.subs);
    }
    if (!batchDepth) {
      flush();
    }
  }
}
```

**审查清单**:
- [ ] 相同值检查存在
- [ ] Dirty 标志正确设置
- [ ] 依赖追踪在 get() 中
- [ ] propagate 在 set() 后调用
- [ ] batchDepth 检查存在

### Signal 使用

```typescript
// ✅ 推荐用法
const count = signal(0);
const doubled = computed(() => count.get() * 2);

effect(() => {
  console.log(doubled.get());
});

count.set(5); // 自动触发更新

// ❌ 需要审查的问题
const count = signal(0);

// 问题 1: 直接访问内部属性
console.log(count.pendingValue);

// 问题 2: 绕过响应式系统
count.pendingValue = 10;

// 问题 3: 未建立依赖
const val = count.pendingValue;
```

**审查清单**:
- [ ] 通过 get()/set() 访问
- [ ] 不直接访问内部属性
- [ ] 在 effect/computed 中使用
- [ ] 避免在外部修改信号

## 🧮 Computed 相关审查

### Computed 实现

```typescript
// 审查要点
class Computed<T, S = T> {
  private _value: T | undefined;
  private _flags: ReactiveFlags = ReactiveFlags.Dirty;
  private _deps?: Link;
  private _getter: (previousValue?: T) => T;
  
  get() {
    // ✓ 依赖追踪
    if (activeSub !== undefined) {
      linkReactiveNode(this, activeSub);
    }
    
    // ✓ 惰性求值
    if (this._flags & ReactiveFlags.Dirty) {
      const oldValue = this._value;
      this._value = this._getter(oldValue);
      this._flags &= ~ReactiveFlags.Dirty;
    }
    
    return this._value!;
  }
}
```

**审查清单**:
- [ ] 惰性求值实现
- [ ] 依赖追踪正确
- [ ] Dirty 标志管理
- [ ] getter 接收 previousValue
- [ ] setter 可选且类型正确

### Computed 使用

```typescript
// ✅ 推荐用法
const fullName = computed(() => {
  return `${firstName.get()} ${lastName.get()}`;
});

// 可编辑的 computed
const editable = computed(
  () => computeValue(),
  (newValue) => updateSource(newValue)
);

// ❌ 需要审查的问题
// 问题 1: 计算中有副作用
const bad = computed(() => {
  apiCall(); // 副作用！
  return value;
});

// 问题 2: 过度嵌套
const veryBad = computed(() => {
  return computed(() => {
    return computed(() => value);
  });
});
```

**审查清单**:
- [ ] getter 是纯函数
- [ ] 无副作用
- [ ] 无嵌套 computed
- [ ] 依赖关系清晰
- [ ] 计算开销合理

## ⚡ Effect 相关审查

### Effect 实现

```typescript
// 审查要点
function effect(fn: () => void): () => void {
  // ✓ 立即执行一次
  fn();
  
  // ✓ 返回清理函数
  return () => {
    cleanup(fn);
  };
}

// ✓ effectScope 支持
function effectScope(fn: () => void): () => void {
  const effects: Array<() => void> = [];
  
  // 收集作用域内的所有 effect
  // ...
  
  // 返回清理函数，停止所有 effect
  return () => {
    effects.forEach(stop => stop());
  };
}
```

**审查清单**:
- [ ] 立即执行
- [ ] 返回清理函数
- [ ] 依赖追踪正确
- [ ] 作用域管理
- [ ] 内存泄漏防护

### Effect 使用

```typescript
// ✅ 推荐用法
const stop = effect(() => {
  document.title = `Count: ${count.get()}`;
});

// 在组件卸载时清理
onUnmounted(() => {
  stop();
});

// 使用 effectScope
const scope = effectScope(() => {
  effect(() => { /* ... */ });
  effect(() => { /* ... */ });
});

// 一次性清理所有
scope();

// ❌ 需要审查的问题
// 问题 1: 忘记清理
effect(() => {
  const id = setInterval(() => {
    console.log(count.get());
  }, 1000);
  // 缺少 clearInterval
});

// 问题 2: 条件依赖
effect(() => {
  if (condition.get()) {
    doSomething(a.get());
  } else {
    doSomething(b.get());
  }
  // 依赖会随 condition 变化，需注意
});
```

**审查清单**:
- [ ] 有清理机制
- [ ] 依赖关系稳定
- [ ] 无无限循环
- [ ] 作用域正确使用
- [ ] 资源及时释放

## 🔄 批量更新审查

### startBatch/endBatch

```typescript
// ✅ 推荐用法
startBatch();
try {
  count.set(1);
  name.set('John');
  age.set(30);
} finally {
  endBatch(); // 确保执行
}

// 嵌套批量更新
startBatch();
a.set(1);

startBatch(); // 深度 +1
b.set(2);
endBatch();   // 深度 -1

c.set(3);
endBatch();   // 深度归零，执行 flush
```

**审查清单**:
- [ ] try-finally 确保 endBatch 调用
- [ ] 成对使用
- [ ] 嵌套深度正确管理
- [ ] 不在批量更新中访问 computed（可能提前触发）

## 🐛 常见错误模式

### 1. 循环依赖

```typescript
// ❌ 错误：循环依赖
const a = computed(() => b.get() + 1);
const b = computed(() => a.get() + 1);

// ✓ 正确：单向依赖链
const a = signal(0);
const b = computed(() => a.get() + 1);
const c = computed(() => b.get() + 1);
```

**审查要点**:
- [ ] 依赖图是有向无环图 (DAG)
- [ ] 无间接循环依赖
- [ ] 动态依赖切换正确

### 2. 内存泄漏

```typescript
// ❌ 错误：未清理 effect
function setup() {
  effect(() => {
    const id = setInterval(() => {
      console.log(count.get());
    }, 1000);
    // 缺少清理
  });
}

// ✓ 正确：带清理
function setup() {
  effect(() => {
    const id = setInterval(() => {
      console.log(count.get());
    }, 1000);
    
    return () => {
      clearInterval(id);
    };
  });
}
```

**审查要点**:
- [ ] 定时器清理
- [ ] 事件监听器移除
- [ ] DOM 引用释放
- [ ] 网络请求取消

### 3. 性能陷阱

```typescript
// ❌ 错误：每次返回新对象
const bad = computed(() => {
  return { value: count.get() }; // 新对象引用
});

// ✓ 正确：缓存或使用原始值
const good = computed(() => {
  return count.get(); // 原始值
});

// 或使用浅比较
const better = computed(() => {
  const val = count.get();
  if (cached && cached.value === val) {
    return cached;
  }
  return { value: val };
});
```

**审查要点**:
- [ ] 避免返回新对象/数组
- [ ] 使用不可变数据
- [ ] 合理使用缓存
- [ ] 避免频繁创建对象
